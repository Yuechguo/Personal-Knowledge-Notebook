### Fix with SMDA issue
``` shell
SDMA FW Update instructions

1. Make note of the current firmware version installed
#>rocm-smi --showfwinfo sdma
 
2. Make a backup copy of the existing firmware:
#>sudo cp /lib/firmware/updates/amdgpu/sdma_4_4_2.bin ~/sdma_4_4_2.bin.bak
 
3. Copy the new bin file into the system firmware update directory:
#> sudo cp ~/sdma_4_4_2.bin /lib/firmware/updates/amdgpu/sdma_4_4_2.bin
 
4. Update the DKMS driver:
    Check dkms version of current driver with:
    #> dkms status | sed 's/,//' | awk {'print $1'} 
    E.g: "amdgpu/6.3.4-1659423.22.04"

    Remove the current dkms driver with:
    #> sudo dkms remove amdgpu/{version from above step} --all

    Rebuild and reinstall dkms driver with:
    #> sudo dkms install amdgpu/{version from above step}
 
    Confirm dkms driver is now installed:
    #> dkms status 
    Should now show that version being again listed as "installed"
 
5. Update initramfs to pick up the new firmware:
     On CentOs, you can do this with: 
      #> sudo dracut --regenerate-all --force
 
6. Reboot and re-load driver after the boot with:
    #> sudo modprobe amdgpu 
 
7.  Check that the new firmware version was installed.
     #> rocm-smi --showfwinfo sdma. 
     For this test firmware, this shows version "21".

sudo mkinitrd /boot/initramfs-`uname -r`.img `uname -r` --force (for rhel)

```

How to reinstall amdgpu dirver:
``` shell
安装amdgpu工具
http://artifactory-cdn.amd.com/artifactory/list/amdgpu-rpm/rhel/
sudo yum install http://artifactory-cdn.amd.com/artifactory/list/amdgpu-rpm/rhel/amdgpu-install-internal-6.3.0.1_8-1.noarch.rpm --nogpgcheck

安装BKC24.12
yum remove --noautoremove *14478*

amdgpu-repo --amdgpu-build=2039227 --rocm-build=compute-rocm-dkms-no-npi-hipclang/14478

driver only: 
amdgpu-install --usecase=dkms 
rocm+driver
amdgpu-install --usecase=rocm

添加黑名单
 grubby --args="modprobe.blacklist=amdgpu" --update-kernel /boot/vmlinuz-5.10.134-13.al8.x86_64

sudo grub2-mkconfig -o /boot/grub2/grub.cfg

新增
grubby --args="nomodeset" --update-kernel /boot/vmlinuz-5.10.134-17.2.al8.x86_64
移除
grubby --remove-args="nomodeset" --update-kernel /boot/vmlinuz-5.10.134-17.2.al8.x86_64
 
reboot
```
